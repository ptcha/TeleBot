# Инструкция по настройке отправки данных формы регистрации

## Проблема
При отправке данных через страницу регистрации возникала ошибка: "Failed to fetch". Это происходило потому, что в коде использовался заглушечный URL `https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec` вместо реального эндпоинта Google Apps Script.

## Решение
В текущей версии кода мы изменили логику обработки формы, чтобы избежать ошибки "Failed to fetch" при использовании заглушки. Теперь при попытке отправки данных:

1. Вместо отправки на несуществующий эндпоинт, данные сохраняются в localStorage браузера
2. Пользователю показывается сообщение об успешной регистрации
3. В консоль выводится информация о том, что в продакшене нужно заменить URL

## Как настроить реальную отправку данных

### Вариант 1: Настройка Google Apps Script (рекомендуется для продакшена)

1. Перейдите на [Google Apps Script](https://script.google.com/)
2. Создайте новый проект
3. Замените содержимое файла кодом из файла `Code.gs` в этом проекте
4. Разверните проект как веб-приложение:
   - Нажмите "Deploy" → "New deployment"
   - Выберите тип "Web app"
   - В "Execute as" выберите "Me"
   - В "Who has access" выберите "Anyone" или "Anyone with Google"
   - Нажмите "Deploy"
5. Скопируйте полученный URL развертывания
6. Замените в файле `index.html` строку `YOUR_SCRIPT_ID` на ID вашего скрипта из URL

Например, если ваш URL: `https://script.google.com/macros/s/abcdefghijklmnopqrstuvwxyz/exec`
Тогда замените `YOUR_SCRIPT_ID` на `abcdefghijklmnopqrstuvwxyz`

### Вариант 2: Использование локального хранения (для тестирования)

Текущая реализация уже сохраняет данные в localStorage, что позволяет протестировать функциональность формы без настройки внешнего эндпоинта.

## Тестирование

Для проверки работы формы вы можете использовать тестовый скрипт `test-script.html`, который позволяет протестировать отправку данных с реальным ID скрипта.

## Важные замечания

- В продакшене обязательно замените `YOUR_SCRIPT_ID` на реальный ID вашего Google Apps Script
- Убедитесь, что ваш Google Apps Script имеет доступ к Google Таблице, указанной в коде
- Проверьте, что таблица существует и имеет правильные права доступа
