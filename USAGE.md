# Использование формы регистрации

После настройки интеграции с Google Apps Script форма регистрации будет отправлять данные напрямую в Google Таблицу.

## Настройка формы

1. Убедитесь, что вы выполнили все шаги из [DEPLOYMENT_GUIDE.md](./DEPLOYMENT_GUIDE.md)
2. Проверьте, что в файле `index.html` заменен placeholder `YOUR_SCRIPT_ID` на реальный ID вашего Google Apps Script
3. Протестируйте интеграцию с помощью файла `test-script.html`

## Проверка работы формы

1. Откройте файл `index.html` в браузере
2. Заполните форму регистрации (имя, фамилия, email)
3. Нажмите "Отправить заявку"
4. Данные должны появиться в вашей Google Таблице
5. Также данные сохраняются в localStorage браузера как резервная копия

## Структура данных

При отправке формы данные записываются в Google Таблицу со следующей структурой:

- Колонка A (ID_reg): Уникальный ID регистрации в формате REG-временная_метка
- Колонка B (ID_masterclass): Уникальный ID мастер-класса в формате MC-временная_метка
- Колонка C (Имя): Имя пользователя из формы
- Колонка D (Фамилия): Фамилия пользователя из формы
- Колонка E (Почта): Email пользователя из формы
- Колонка F (Статус): Статус регистрации (по умолчанию "Записан")

## Обработка ошибок

- Если Google Apps Script недоступен, данные сохраняются в localStorage браузера
- Пользователь получает уведомление о статусе операции
- Все ошибки логируются в консоль браузера для отладки

## Локальное хранение данных

Для дополнительной надежности все данные также сохраняются в localStorage браузера в формате JSON. Вы можете получить доступ к этим данным через консоль браузера:

```javascript
JSON.parse(localStorage.getItem('registrations'))
```

## Тестирование

Используйте файл `test-script.html` для проверки работы интеграции перед использованием основной формы регистрации.
